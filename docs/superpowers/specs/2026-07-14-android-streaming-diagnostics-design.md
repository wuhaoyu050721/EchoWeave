# Android 流式诊断页设计

## 1. 目标

在现有 uni-app Vue 3 项目中增加一个仅用于 Android App 的独立诊断页，验证 `uni.request` 的 POST 流式响应、`RequestTask.onChunkReceived`、SSE 解析、主动停止以及 App 前后台切换行为。

当前可使用 Android 17 x86_64 模拟器完成第一轮验证。模拟器通过不代表所有真机环境均通过，因此界面分别展示“模拟器已验证”和“真机未验证”状态。

本阶段不替换现有浏览器聊天传输层，也不实现 SQLite、Android Keystore、PHP/MySQL 或云端同步。

## 2. 页面入口与布局

- 在现有设置页增加“Android 流式诊断”入口。
- 新增独立页面 `pages/android-diagnostics/index`。
- 页面沿用现有应用配色和紧凑移动端布局，不改变聊天页布局。
- H5 打开该页面时显示“仅 Android App 支持流式诊断”，允许查看说明但禁用开始按钮。

页面分为四个区域：

1. 请求配置：接口地址、API Key、模型、测试提示词和请求超时。
2. 操作栏：开始、停止、清空日志和导出日志。
3. 状态摘要：运行环境、请求状态、首块耗时、总耗时、分块数、字节数、SSE 事件数以及结束原因。
4. 诊断日志：按时间顺序展示生命周期、原始分块摘要和 SSE 解析结果。

## 3. 模块边界

### 3.1 AndroidDiagnosticTransport

负责：

- 使用 `uni.request` 发起 OpenAI 兼容的 `/chat/completions` POST 请求。
- 开启流式响应并订阅 `RequestTask.onChunkReceived`。
- 将每个原始分块交给诊断记录器和 SSE 解析器。
- 调用 `RequestTask.abort()` 停止请求。
- 在停止后忽略迟到的业务事件，同时保留“收到迟到分块”的诊断记录。

不负责保存接口配置，不直接修改聊天记录。

### 3.2 DiagnosticSseSession

复用现有 SSE 解析规则，并补充字节流到 UTF-8 文本的增量解码。必须覆盖：

- 一个分块包含多个 SSE 事件。
- 一个 SSE 事件跨多个分块。
- 中文 UTF-8 字符跨分块。
- 多行 `data:`。
- `[DONE]`。
- 网络正常结束但没有 `[DONE]`。
- 停止后到达的迟到分块。

### 3.3 DiagnosticLogStore

只在页面内存中保存本次诊断日志。每条日志包含相对时间、类别、分块序号、分块字节数和脱敏后的摘要。

禁止记录：

- API Key 原文。
- `Authorization` 请求头原文。
- 用户未明确选择导出的完整响应内容。

导出文件默认只包含诊断元数据和截断后的内容摘要。

### 3.4 AppLifecycleRecorder

监听 App 前台、后台和页面卸载事件，将时间点写入诊断日志。页面卸载时必须停止尚未结束的请求并移除监听器。

## 4. 请求与数据流

1. 用户填写接口地址、API Key、模型和测试提示词。
2. 页面只在内存中持有 API Key，不写入 IndexedDB、SQLite 或应用设置。
3. 点击开始后校验必填字段并规范化 OpenAI 兼容地址。
4. 创建请求会话并写入开始时间。
5. 每个原始分块先记录字节数和到达时间，再进行 UTF-8 增量解码和 SSE 解析。
6. 解析出的内容增量显示在预览区，同时记录事件数量和结束原因。
7. 收到 `[DONE]`、请求正常完成、发生错误或用户停止时结束会话。
8. 用户可以清空内存日志，或导出已经脱敏的 JSON 诊断文件。

## 5. 状态与错误处理

请求状态为：

- `idle`
- `connecting`
- `streaming`
- `completed`
- `aborted`
- `failed`

同一时间只允许一个诊断请求。运行中再次点击开始不会创建第二个请求。

错误分类至少包含：

- 地址或参数无效。
- DNS、TLS 或网络不可达。
- HTTP 非成功状态。
- 响应不是可解析的 SSE。
- UTF-8 或 JSON 数据异常。
- 平台不支持 `onChunkReceived`。
- 请求超时。
- 用户主动停止。

错误信息必须保留可操作的上下文，但不得包含 API Key。

## 6. 验证方案

### 6.1 自动测试

- 地址规范化和请求体构造。
- UTF-8 中文跨字节分块。
- 一个分块多事件和一个事件多分块。
- `[DONE]` 与无 `[DONE]` 完成。
- 停止后迟到分块不再更新内容。
- 日志和导出内容不包含 API Key。

### 6.2 Android 模拟器

使用当前 Android 17 x86_64 模拟器验证：

- 页面可从设置进入。
- POST 流式分块回调真实触发。
- 内容逐段显示而不是请求结束后一次显示。
- 停止按钮能中止请求。
- 切换前后台后生命周期日志正确。
- 页面退出时请求和监听器被清理。
- 导出的日志不包含 API Key。

用户可以使用自己的 OpenAI 兼容接口完成联网验证。本项目不内置或记录真实密钥。

### 6.3 后续真机补充验证

真机仅补充验证模拟器无法代表的行为：

- 厂商系统后台限制。
- 锁屏和恢复。
- Wi-Fi 与移动网络切换。
- 弱网和系统回收。

未完成真机验证前，不宣称 Android 传输层已覆盖所有设备。

## 7. 完成标准

- 独立诊断页和设置入口可用。
- H5 不报错且明确禁用 Android 专属操作。
- 自动测试与生产构建通过。
- Android 模拟器完成流式、停止和生命周期验证。
- API Key 未被持久化、记录或导出。
- 验证结果足以决定正式 `ChatTransport` 使用 `uni.request`，还是升级为 UTS/OkHttp 实现。

# Four-Screen UI Redesign Design

**Date:** 2026-07-15

## Goal

按用户提供的四屏效果图改造会话首页、接口页、设置概览和设置详情，保留现有数据库、加密、聊天、附件、云备份和诊断功能。

## Reference

- 效果图：`C:/Users/Administrator/AppData/Local/Temp/codex-clipboard-93528754-ed2a-4af5-b4b1-a0c1cc3555ab.png`
- 浏览器视觉验收视口：`390 x 844`
- Android 验收：当前 HBuilderX Android 模拟器实际视口
- 不添加效果图中的假手机状态栏，继续使用系统原生状态栏和安全区。

## Scope

### Included

1. 会话首页：保留现有布局，只调整配色、边框、阴影和激活态。
2. 接口页：按效果图重排顶栏、接口卡片、表单和操作按钮。
3. 设置概览：按效果图展示基础设置卡片、云备份卡片和安全入口。
4. 设置详情：新增设置内部详情状态，展示安全和应用信息分组。

### Excluded

- 聊天页布局和样式。
- Android 流式诊断页布局和样式。
- 数据表、备份格式、后端 API 或请求协议变更。
- 效果图首页中的“最近会话 / 已置顶”筛选与编辑悬浮按钮，因用户明确要求首页保留现有布局。

## Visual System

- 主色：`#1F6FCB`
- 深主色：`#155DB6`
- 浅主色：`#E8F1FD`
- 页面背景：`#F5F7FA`
- 卡片背景：`#FFFFFF`
- 主文字：`#172033`
- 次文字：`#758196`
- 边框：`#DCE3EC`
- 成功：`#36B52A`
- 危险：`#E5484D`
- 卡片圆角最大 `8px`，使用轻边框和低对比阴影。
- 标题栏使用纯色蓝底和白色图标/文字，不使用渐变。
- 底部导航使用白底，当前项为蓝色，其余项为中性灰。
- 继续使用已修复的 App 本地字体图标和 PNG 接口 Logo，不恢复 SVG 组件。

## Screen 1: Conversations

- 保留现有顶栏、搜索框、会话列表、新建会话悬浮按钮和三项底部导航的位置与交互。
- 顶栏改为蓝底白字；搜索框使用浅灰蓝背景。
- 会话行改为白色卡片化表现，但不增加新的信息层级或操作。
- 会话图标使用浅蓝色容器，新建按钮使用主色蓝。

## Screen 2: Providers

- 顶栏左侧为“接口”，右侧为带加号的“添加接口”。
- 接口列表改为独立白色卡片，展示 PNG Logo、名称、基础地址、默认模型和蓝色选中标记。
- 表单使用“编辑接口 / 添加接口”分区标题和白色卡片，字段按效果图纵向堆叠。
- API 密钥保留显示/隐藏按钮。
- “获取模型列表”、“测试连接”和“保存”放在卡片底部同一操作区，保存为主按钮。
- 保留现有加密密钥、模型列表、连接测试、新增、修改和删除逻辑。

## Screen 3: Settings Overview

- 顶栏左侧为“设置”，右侧更多按钮进入设置详情。
- 第一张白色卡片包含系统提示词、本地数据、导入与导出、云端备份四行。
- 系统提示词编辑器仍在当前行下方展开，使用蓝色开关和主操作按钮。
- 云备份使用独立卡片，根据实际登录状态显示登录/注册或账号、服务器、同步密码、自动备份和备份操作。
- 已登录时使用绿色小状态点和“已启用”文案；危险操作使用红色。
- 概览页底部展示“安全”预览入口，点击 API 密钥管理行进入设置详情。

## Screen 4: Settings Details

- 顶栏左侧为返回按钮和“设置”，返回设置概览。
- “安全”分组包含 API 密钥管理和应用锁。
- API 密钥管理继续跳转到接口页。
- 应用锁保留现有开关状态与持久化。
- “应用信息”分组包含 Android 流式诊断、关于应用、检查更新和帮助与反馈。
- 诊断入口仍打开现有 `pages/android-diagnostics/index` 路由，其页面本身不改造。
- 底部导航保留显示，“设置”仍为蓝色激活态。

## Navigation And State

- 在主页本地 UI 状态中新增 `settingsView: 'overview' | 'details'`。
- 点击设置底部导航时默认进入 `overview`。
- 设置概览右上角更多按钮和安全预览行将状态设为 `details`。
- 详情页返回按钮将状态恢复为 `overview`。
- 离开设置页后再返回时，从概览开始，避免用户被留在次级界面。

## Behavior Preservation

- 所有现有按钮继续调用原有 methods，不修改服务层接口。
- 云备份根据 `cloudConnected`、`cloudBusy`、`autoBackupEnabled` 和 `cloudBackupStatus` 显示实际状态。
- 接口表单继续屏蔽已保存密钥，空密钥不覆盖现有密文。
- 错误横幅、Toast、导入/导出弹窗和系统提示词编辑器继续工作。

## Responsive And Platform Requirements

- 固定顶栏和底部导航高度，中间内容单独滚动。
- 表单字段、卡片和操作按钮在 `360px` 与 `390px` 宽度下不溢出、不重叠。
- 文本超长时截断或换行，不改变固定图标和按钮尺寸。
- 底部操作不被系统手势区遮挡。

## Testing And Visual QA

- 为设置概览/详情状态转换增加纯 UI 测试。
- 增加源码级 UI 结构回归测试，确认四屏必需标题、卡片、操作与入口存在。
- 运行 `npm test` 和 `npm run build`。
- 在浏览器 `390 x 844` 视口截取四个对应状态。
- 恢复 Android 模拟器后，冷启动并截图验证图标、Logo、安全区、滚动和交互。
- 对照效果图检查标题栏、卡片边界、间距、表单密度、蓝色激活态和底部导航。
- 修复所有 P0、P1 和 P2 差异后才完成交付。

## Architecture

- 采用渐进改造：保留 `pages/index/index.vue` 的现有服务绑定和 methods，只重排四个目标视图的模板与样式。
- 设置概览/详情是同一页内部状态，不增加新路由。
- 只在纯 UI 状态需要可测试时修改 `src/ui-state.js`，不引入新依赖。
